# خارطة طريق Voltify — احترافية وأمان واكتمال المنصة

## ما أصبح جاهزاً الآن

| الميزة | الحالة |
|--------|--------|
| موقع عملاء + شحن ألعاب + صيانة + تتبع | ✅ |
| لوحة تحكم API (`admin.html`) | ✅ |
| إدارة الخدمات (صورة، وصف، 3 لغات، قريباً، ترتيب) | ✅ جديد |
| منتجات / إعلانات / معرض / مستخدمون / صلاحيات | ✅ |
| نشر برابط واحد | ✅ (انظر `DEPLOY-AR.md`) |

---

## أين تضيف الخدمات؟

1. شغّل السيرفر: `backend\run-dev.bat`
2. افتح: **http://localhost:4000/admin.html**
3. من القائمة: **🔧 الخدمات**
4. املأ النموذج:
   - **صورة:** اختر ملف → «رفع الصورة» → «حفظ الخدمة»
   - **قريباً:** فعّل عند البداية → تظهر شارة ذهبية على البطاقة
   - عند الجاهزية: من البطاقة أسفل اضغط **«تفعيل (إزالة قريباً)»**
5. حدّث الموقع (`index.html` عبر نفس السيرفر) لترى التغيير فوراً

## ما يزال لجعل المنصة احترافية (بالأولوية)

### أولوية عالية — أمان واستقرار

1. **قاعدة بيانات حقيقية** (PostgreSQL أو MongoDB) بدل ملف JSON — ضروري عند كثرة الطلبات والنسخ الاحتياطي.
2. **HTTPS إجباري** على الدومين + تغيير `JWT_SECRET` وكلمة الأدمن قبل النشر.
3. **نسخ احتياطي يومي** لـ `backend/data/` (store.json + uploads).
4. **حدّ معدّل الطلبات** (rate limit) على تسجيل الدخول وواجهات `public` لمنع الإساءة.
5. **سجل تدقيق (Audit log):** من عدّل/حذف طلباً أو خدمة وبأي وقت.

### أولوية متوسطة — تجربة العميل

6. **نموذج طلب مخصص لشحن الرصيد** (رقم الهاتف + المشغّل + المبلغ) مرتبط بـ `order_type: topup`.
7. **إشعارات واتساب/تيليجرام تلقائية** عند طلب جديد (Webhook أو بوت).
8. **لوحة إحصائيات متقدمة** (مبيعات يومية، أكثر لعبة، أكثر خدمة).
9. **صفحة حالة الخدمة** (الموقع يعمل / صيانة).
10. **بريد إلكتروني** لتأكيد الطلب ورمز التتبع.

### أولوية متوسطة — إدارة المحتوى

11. **محرر صفحات** (نصوص الهيرو، العروض) من لوحة التحكم بدون تعديل HTML.
12. **جدولة الإعلانات** (يبدأ/ينتهي في تاريخ محدد).
13. **نسخ متعددة للصور** (ضغط تلقائي WebP).
14. **سحب وإفلات** لترتيب الخدمات والمنتجات.

### أولوية منخفضة — نمو الأعمال

15. **بوابة دفع** (CIB، BaridiMob، PayPal) مع تأكيد تلقائي.
16. **حسابات عملاء** (تسجيل، سجل طلبات، نقاط ولاء).
17. **تطبيق PWA** (تثبيت على الجوال).
18. **لوحة وكيل** (موزعون يشحنون تحت حسابك).
19. **تقارير ضريبية / فواتير PDF** رسمية.

## الأمان — قائمة تحقق سريعة

- [ ] كلمة أدمن قوية + لا تشاركها
- [ ] `JWT_SECRET` عشوائي 32+ حرفاً في `.env`
- [ ] لا ترفع `.env` إلى GitHub
- [ ] لوحة التحكم عبر HTTPS فقط
- [ ] أدوار: مشاهد للموظف، محرر للعمليات، مدير عام لك فقط
- [ ] مراجعة الطلبات المشبوهة (نفس الرقم عشرات المرات)

## لتستطيع «إضافة أي شيء» لاحقاً

الهيكل الحالي يدعم التوسع هكذا:

| تريد إضافة | أين الآن | التطوير المقترح |
|------------|----------|------------------|
| خدمة جديدة | لوحة → الخدمات | جاهز |
| لعبة / باقة | لوحة → المنتجات | جاهز |
| شحن رصيد | منتجات `topup` + خدمة `topup` | نموذج طلب مخصص |
| صورة معرض | لوحة → المعرض | جاهز |
| إعلان | لوحة → الإعلانات | جاهز |
| صفحة جديدة | تعديل `index.html` | محرر صفحات |
| تطبيق جوال | — | API جاهز للربط |

## الخطوة التالية المقترحة (عندما تريد)

1. نموذج **شحن رصيد** كامل من الموقع إلى لوحة التحكم.
2. **إشعار واتساب** عند كل طلب جديد.
3. **نشر على Render** برابط واحد (راجع `DEPLOY-AR.md`).

إذا حددت ما تريد أولاً (مثلاً: شحن الرصيد + واتساب)، يمكن بناؤه في الجلسة القادمة خطوة بخطوة.
